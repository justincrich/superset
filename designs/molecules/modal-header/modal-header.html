Review Plan

Overview

This plan proposes a three-phase migration of the relay tunnel reconnect logic. Phase one isolates the transport layer behind a stable interface. Phase two introduces offline-queue persistence so messages survive network gaps. Phase three enables full duplex streaming with backpressure.

Phase 1 — Transport Isolation

Extract the current inline WebSocket calls into a RelayTransport interface. Introduce two concrete implementations: WebSocketTransport (production) and LocalTransport (test). All consumer code becomes transport-agnostic.

Phase 2 — Offline Queue

Persist outbound messages to a local SQLite queue before handoff to the transport layer. On reconnect, drain the queue in FIFO order with exponential back-off. Duplicate detection uses a client-assigned msgId correlated against the server-side idempotency log.

Phase 3 — Streaming

Replace the request–response model with a full-duplex stream. Apply backpressure via a bounded token-bucket on the sender side. The receiver signals demand through WINDOW_UPDATE frames — mirroring HTTP/2 flow control semantics at the application layer.

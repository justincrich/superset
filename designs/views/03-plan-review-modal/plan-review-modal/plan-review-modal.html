Billing Refactor Plan

Phase 1: Extract router

Move billing.ts to a dedicated tRPC router file under packages/trpc/src/routers/. The current inline routes mix REST handlers with the data layer; extracting them gives us reusable Zod input schemas for both client and server validation.

Phase 2: Add procedures

Define getInvoice, listInvoices, and updateInvoice procedures with strict Zod input schemas. Procedures will reuse the existing db.invoice.* helpers but with type-safe wrappers.

Phase 3: Migrate REST callsites

Update the REST callsites in apps/web/src/billing/* to use trpc.billing.*.useMutation() and useQuery() hooks. This removes the manual fetch + JSON parsing layer.

Phase 4: Update tests

Migrate tests in apps/web/test/billing.test.ts to mock the tRPC procedures via createMockTRPCContext.

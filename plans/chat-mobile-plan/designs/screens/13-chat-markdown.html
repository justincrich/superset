chat_sessions — Electric-shape ready

The new schema separates lightweight session metadata from message bodies, so we can sync the list cheaply while keeping the bulky thread on the host.

  1. List shape — chat_sessions rows replicate via Electric.
  2. Messages — stay in host-runtime memory; fetched on demand.
  3. Cursor — opaque lastUpdatedAt per row, used for resume.

Schema

sql-- packages/db/src/schema/chat-sessions.ts
CREATE TABLE chat_sessions (
  id          uuid      PRIMARY KEY,
  org_id      uuid      NOT NULL,
  host_id     uuid      NOT NULL,
  workspace_id uuid     NOT NULL,
  title       text      NOT NULL,
  status      session_status NOT NULL,
  last_active timestamptz NOT NULL
);
Heads up: session_status is a postgres enum — adding a value is a migration of its own.
